returns a copy of the mesh_prototype object created in the solve_child method, the reason we use this is because when we push a mesh_prototype object into the meshes vector it is stored by reference, and when the object goes out of the destructor is called, resulting in us havinmg to fetch the data from the RAM again To avoid that we create a copy of the class, and since OpenGl bindings are universal the VAO's are not los, Also the copy is not created by calling the constructor again, but by replicating memory, so we don't need to worry about extra VAO's
